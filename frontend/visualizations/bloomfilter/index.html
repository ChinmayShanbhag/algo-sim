Understanding Bloom Filters

What is a Bloom Filter? üå∏

A Bloom Filter is a space-efficient probabilistic data structure that tests whether an element is a member of a set. It can tell you:

  • "Definitely NOT in the set" (100% accurate)
  • "Probably in the set" (might be a false positive)

How It Works:

1. Bit Array üî¢

A fixed-size array of bits (all initially 0). In this demo: 64 bits.

2. Hash Functions üî®

Multiple independent hash functions (here: 3). Each maps an item to a position in the bit array.

3. Adding an Item ‚ûï

  1. Hash the item with each hash function
  2. Set the bits at those positions to 1
  3. Example: "apple" ‚Üí Hash1(15), Hash2(42), Hash3(7) ‚Üí Set bits 15, 42, 7 to 1

4. Checking an Item ‚úÖ

  1. Hash the item with each hash function
  2. Check if ALL those bits are 1
  3. If ANY bit is 0 ‚Üí Definitely NOT in set
  4. If ALL bits are 1 ‚Üí Probably in set (might be false positive)

False Positives üö®

A false positive occurs when:

  • An item was NOT added to the filter
  • But its hash positions were set by OTHER items
  • So the filter says "probably yes" incorrectly

Example: If "apple" sets bits [15, 42, 7] and "banana" sets bits [15, 23, 42], then checking "cherry" which hashes to [15, 42, 7] will return "probably yes" even though it wasn't added!

Key Properties:

  • Space Efficient: Uses much less space than storing all items
  • Fast: O(k) time for add/check (k = number of hash functions)
  • No False Negatives: If it says "not in set", it's 100% correct
  • Can't Remove Items: Standard Bloom Filters don't support deletion
  • False Positive Rate: Increases as more items are added (more bits set)

Trade-offs:

  • Larger bit array: Fewer false positives, more memory
  • More hash functions: Fewer false positives, slower operations
  • More items added: Higher false positive rate

Real-World Uses:

  • Chrome: Checks malicious URLs (millions of URLs in tiny space)
  • Medium: "Have you read this article?" recommendations
  • Bitcoin: Simplified Payment Verification
  • Cassandra/HBase: Avoid disk reads for non-existent keys
  • Akamai: Web cache filtering
  • Spell Checkers: Quick dictionary lookups

Try It:

  1. ADD several items (e.g., "apple", "banana", "cherry")
  2. Watch how hash functions set different bits
  3. CHECK items you added ‚Üí "probably yes" (true positive)
  4. CHECK items you didn't add ‚Üí might get false positives!
  5. Add more items and watch the false positive rate increase

How We Simulate Bloom Filters

Data Structure
STRUCTURE BloomFilter:
    bitArray = array[32] of boolean  // 32 bits (all initially false)
    numHashes = 3                    // Number of hash functions
    itemsAdded = 0                   // Track items count
Hash Functions (FNV-1a with Seeds)
FUNCTION getHashPositions(item):
    positions = []
    
    FOR i = 0 TO numHashes-1 DO
        // Use hash function with different seed for each position
        seed = concatenate(item, ":", i)
        hashValue = FNV1a_Hash(seed)
        
        // Map hash to bit array position (0-31)
        position = hashValue MOD 32
        positions.append(position)
    END FOR
    
    RETURN positions
END FUNCTION

EXAMPLE for item "apple" with 3 hash functions:
    Hash 0: FNV("apple:0") MOD 32 = 7
    Hash 1: FNV("apple:1") MOD 32 = 15
    Hash 2: FNV("apple:2") MOD 32 = 23
    ‚Üí Sets bits at positions 7, 15, 23
Add Operation
FUNCTION Add(item):
    // Get all hash positions for this item
    positions = getHashPositions(item)
    
    // Set all corresponding bits to true
    FOR EACH position IN positions DO
        bitArray[position] = true
    END FOR
    
    itemsAdded = itemsAdded + 1
END FUNCTION

Time Complexity: O(k) where k = number of hash functions (k=3)
Space Complexity: O(1) - fixed size bit array

EXAMPLE: Add("apple")
    Step 1: Calculate positions ‚Üí [7, 15, 23]
    Step 2: Set bitArray[7] = true
    Step 3: Set bitArray[15] = true
    Step 4: Set bitArray[23] = true
Check Operation
FUNCTION Check(item):
    // Get all hash positions for this item
    positions = getHashPositions(item)
    
    // Check if ALL corresponding bits are set
    FOR EACH position IN positions DO
        IF bitArray[position] == false THEN
            RETURN false  // Definitely NOT in set
        END IF
    END FOR
    
    RETURN true  // PROBABLY in set (might be false positive)
END FUNCTION

Time Complexity: O(k)

KEY PROPERTIES:
    ‚úÖ No False Negatives: If returns false, item definitely NOT in set
    ‚ö†Ô∏è False Positives Possible: If returns true, item MIGHT be in set

EXAMPLE: Check("banana")
    Step 1: Calculate positions ‚Üí [2, 15, 28]
    Step 2: Check bitArray[2] ‚Üí true
    Step 3: Check bitArray[15] ‚Üí true (was set by "apple"!)
    Step 4: Check bitArray[28] ‚Üí false
    Result: FALSE (definitely not in set)
False Positive Probability
FUNCTION FalsePositiveRate():
    // Count how many bits are set to true
    bitsSet = 0
    FOR i = 0 TO 31 DO
        IF bitArray[i] == true THEN
            bitsSet = bitsSet + 1
        END IF
    END FOR
    
    // Calculate probability
    fillRatio = bitsSet / 32
    fpRate = fillRatio ^ numHashes
    
    RETURN fpRate * 100  // Return as percentage
END FUNCTION

MATHEMATICAL FORMULA:
    FP Rate ‚âà (1 - e^(-kn/m))^k
    where:
        k = number of hash functions (3)
        n = number of items inserted
        m = number of bits in array (32)

EXAMPLES with our configuration (m=32, k=3):
    0 items  ‚Üí ~0% FP rate
    5 items  ‚Üí ~8% FP rate
    10 items ‚Üí ~32% FP rate
    15 items ‚Üí ~60% FP rate
    
As more items are added, more bits get set to true,
increasing the chance of false positives!
